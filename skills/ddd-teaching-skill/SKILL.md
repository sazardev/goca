# DDD Teaching Skill

## Description
Teaches Domain-Driven Design concepts with focus on Go implementation patterns including Value Objects, test fixtures, mocking strategies, Docker Compose testing setup, HTTPie for end-to-end validation, structured logging, and execution traceability.

## Implementation
- Apply DDD principles in Go with clear separation of domain, application, and infrastructure layers
- Use Value Objects for business rules and validation
- Implement test fixtures with Go testing patterns
- Setup mocking for unit tests
- Use Docker Compose for service testing
- Validate end-to-end with HTTPie
- Implement structured logging and execution tracing
